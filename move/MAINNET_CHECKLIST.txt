Mainnet cutover — чекліст (ffl-moves)
====================================

Офіційні джерела Movement
-------------------------
• Movement CLI: https://docs.movementnetwork.xyz/devs/movementcli
• Network Endpoints (RPC, Chain ID, explorers): https://docs.movementnetwork.xyz/devs/networkEndpoints
• Mainnet: Chain ID **126**, primary RPC **https://mainnet.movementnetwork.xyz/v1**
• Testnet: Chain ID **250**, RPC **https://testnet.movementnetwork.xyz/v1**
• Статус мережі: https://status.movementnetwork.xyz/

Код і збірка
------------
[ ] cd move/fantasy-epl-contract && movement move compile --dev
[ ] movement move test --dev   (усі тести PASS)
[ ] npm run build               (у корені репо)

Перевірка змінних (локально + Vercel)
--------------------------------------
[ ] npm run preflight:mainnet   (заповнений .env.local з MAINNET значеннями)
[ ] NEXT_PUBLIC_MOVEMENT_RPC_URL = **https://mainnet.movementnetwork.xyz/v1** (або партнерський RPC з Network Endpoints)
[ ] NEXT_PUBLIC_MODULE_ADDRESS = адреса пакета **після** publish на mainnet
[ ] NEXT_PUBLIC_MODULE_NAME = fantasy_epl
[ ] У Vercel (Production) ті самі змінні → **Redeploy** після зміни env
